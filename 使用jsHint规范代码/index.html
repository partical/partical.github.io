使用jsHint规范js代码

工作中主要使用eclipse来进行java Web开发,js的代码检测使用jshint来进行，这里主要写写jshint 的配置项。
***
`
{
"indent" : 4,// 使用4个空格来缩进
"camelcase" : true, // 变量名必须使用驼峰写法或者大写
"latedef" : "nofunc", //函数定义可以在使用之后
"newcap" : true,// new 之后的首字母必须大写
"curly" : true,//语句块必须使用大括号，不能写成单行
"eqeqeq" : false,//判断相等的时候不强制使用===
"eqnull" : true,//判断为null的时候可以使用 ==
"strict" : false,//不进入严格模式
"undef" : true,//使用的变量未定义
"unused" : true,//定义的变量未使用
"laxcomma" : true,//逗号写在行末，而不是行首
"asi" : false, //不使用js自动补全机制
"boss" : false, //条件判断中不允许直接赋值
"expr" : true,//表达式可以直接运行
"sub" : true, //对象访问支持点号和中括号
"quotmark" : "single", // 使用单引号, true ： 表示引号互相匹配即可，double: 使用双引号
"loopfunc" : false, // 在循环中不定义函数
"lastsemic" : false, //单行不省略语句的分号
"funcscope" : false, //将非函数块也当作块来处理
"esnext" : false, //不支持支持es6
"noyield" : true, // 不支持es6的使用
"validthis" : false, //function里面不使用this
"freeze" : true, //对native objects 的原型增加方法会提示错误
"forin" : false, //对for in 使用不要求必须检测key是否是object自身的属性(obj.hasOwnProperty(key))
"immed" : true, //立即执行的函数需要使用括号包裹
"noarg" : true,// 禁止使用arguments.caller 和 arguments.callee
"trailing" : true, //禁止代码末尾出现空行
"maxdepth" : 3, //函数的最大嵌套层数
"maxparams" : 5, //函数参数的最大个数
"maxstatements" : 20, //函数的最大语句个数，内部函数声明只当作一条语句
"debug" : false, //代码中有debugger会给出警告
"evil" : false, // 不允许代码中出现eval
"laxbreak" : false, //不安全的换行会给出警告
"browser" : true, //可以使用浏览器的全局变量，比如document
"devel" : true ,// 可以使用调试的全局变量，比如alert console
"jquery": true, //可以使用jquery的全局变量
"predef": [ "sz" ], //全局变量
"globals" : {
    "define" : false //全局的变量只读
}

}
`

***
以上注释的意思都是配置项生效的意思。详情可以参考[这里](http://jshint.com/docs/options/#options), 汉化参考[这里](https://github.com/mytcer/jshint-docs-cn/blob/master/enforcing_options.md)。
